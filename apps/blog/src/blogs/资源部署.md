当我有自己的云服务器后，如何把自己的前端项目部署到云服务器上呢？方便直接在公网上访问  
以下是部署前端代码到阿里云服务器的步骤： 
<!-- more --> 
1. 通过Xshell连接到服务器，输入主机名、端口号、用户名、密码
2. 安装web服务器(一般是nginx)
```
# Ubuntu/Debian
sudo apt-get update
sudo apt-get -y install nginx
```
3. 前端把项目打包成静态文件
```
npm run build
```
4. 把打包后的静态文件上传到服务器  
一般是把静态文件放到/var/www/html里，如果你想放多个项目，建议每个项目创建一个文件夹，比如/var/www/html/project1，/var/www/html/project2

5. 配置nginx，让nginx指向打包后的静态文件
找到nignx的配置文件，一般是/etc/nginx/nginx.conf，修改server块的配置
```
http {
    ...
    server {
        listen 80;
        # 直接使用IP地址作为server_name
        server_name ip地址或域名; 

        # 静态文件路径配置（假设代码在/var/www/html目录）
        root /var/www/html;
        index index.html;

        location / {
            try_files $uri $uri/ /index.html;
        }
    }
    ...
}
  
```
6. 重启nginx
```
sudo systemctl restart nginx
```
7. 访问服务器的IP地址，即可访问到前端项目 

#### 如果我想在云服务器上部署多个前端项目呢，比如demo1，demo2。
1. 把demo1和demo2的代码打包成静态文件
2. 把打包后的静态文件上传到服务器，比如/var/www/html/demo1，/var/www/html/demo2
3. 配置nginx
```
http {
   ...
    server {
    listen 80;
    server_name 47.98.56.114;

    # 项目1配置
    location /demo1 {
        alias /var/www/html/demo1;
        index index.html;
        try_files $uri $uri/ /demo1/index.html;
    }

    # 项目2配置
    location /demo2 {
        alias /var/www/html/demo2;
        index index.html;
        try_files $uri $uri/ /demo2/index.html;
    }
}
}
```
4. 重启nginx
5. 访问对应的路径，即可访问到对应的项目  
ip地址/demo1  
ip地址/demo2

`注意：如果访问时，无法打开页面，可能是云服务器没有配置规则`
1. 登录阿里云ECS控制台
2. 找到您的实例 -> 安全组 -> 管理规则 -> 快速添加
3. 将端口范围是HTTP(80)和HTTPS(443)的规则添加到安全组中
